# 🎮 Game Dev e Videogiochi

## La filosofia open source nel game development

Nel club crediamo che la creatività non debba avere barriere. Per questo, in tutti gli eventi legati allo sviluppo di videogiochi, adottiamo una regola fondamentale:

> 🎮 **Regola Open Source First:** tutto ciò che viene realizzato durante gli eventi del club deve utilizzare esclusivamente strumenti **gratuiti e open source**. Software libero, risorse libere, condivisione della conoscenza.

### Eccezioni alla regola

Se hai un'idea che richiederebbe l'uso di software o risorse non open source (es. un motore grafico proprietario, un plugin a pagamento, asset con licenza non libera), **puoi comunque fare richiesta**.

La procedura è semplice:
1. **Contatta i giudici dell'evento** (gli organizzatori) prima dell'inizio
2. **Spiega perché** lo strumento non open source è necessario
3. **Descrivi il progetto** e come verrà utilizzato
4. I giudici valutano e danno una risposta: *approvato* o *declinato*

## Godot Engine

**Godot** è il nostro motore di riferimento: completamente open source, leggero, potente e con una community in forte crescita.

### Perché Godot

| Vantaggio | Descrizione |
|-----------|-------------|
| Open source | Gratuito, senza royalty, codice modificabile |
| Multipiattaforma | Pubblica su Windows, Linux, macOS, Android, iOS, Web |
| Linguaggi flessibili | GDScript (simile a Python), C#, C++, VisualScript |
| Leggero | Non richiede hardware potente |
| 2D e 3D | Ottimo per entrambi, con strumenti dedicati |

### Iniziare con Godot

1. **Scarica Godot** dal sito ufficiale (versione stabile)
2. **Segui un tutorial** — ne consigliamo uno sul nostro Discord
3. **Crea il tuo primo progetto**: un semplice "Clicca e raccogli"
4. **Sperimenta** con nodi, scene e segnali

## Game Jam — Guida completa

La **Game Jam Open Source** è il nostro evento principale di sviluppo: 48 ore per creare un videogioco usando solo strumenti open source. Si svolge **completamente online** su Discord.

### Come funziona

1. **Il tema** viene votato dalla community Discord nelle settimane precedenti
2. **Si parte** il venerdì sera (o sabato mattina) in voice chat
3. **Si sviluppa** in 48 ore, da soli o in team
4. **Si presenta** il gioco alla fine, con gameplay e spiegazione del processo
5. **I giudici** (gli organizzatori) valutano i giochi e assegnano premi goliardici

### Regole della Game Jam

| Regola | Dettaglio |
|--------|-----------|
| Open source first | Tutti gli strumenti usati devono essere open source. Eccezioni vanno approvate dai giudici prima dell'inizio |
| Tutto dal vivo | Lo sviluppo avviene durante l'evento (non si portano giochi già iniziati) |
| Asset liberi | Se usi grafica, musica o suoni, devono avere licenza open source o Creative Commons |
| Condivisione | Alla fine, il codice e gli asset vengono condivisi su GitHub (licenza open source) |
| Team | Puoi lavorare da solo o in gruppo, ma tutti i membri devono essere presenti su Discord |

### Cosa preparare prima

- PC con Godot installato
- Cuffie e microfono per Discord
- Taccuino per appunti e idee
- Snack e bevande a portata di mano

## Gioca il mio gioco

Un evento ricorrente in cui i membri che stanno sviluppando qualcosa lo fanno provare agli altri per ricevere feedback. Si svolge in voice chat su Discord con condivisione schermo.

### Come funziona

- Ogni membro prenota uno slot di 15-20 minuti
- Mostra il gioco (anche se incompleto)
- Gli altri giocano (se possibile) e danno feedback costruttivo
- Si discute insieme di miglioramenti

## Game Jam Terminale

Una sfida diversa: fare un gioco in **2 ore** usando **solo il terminale**.

### Regole

| Regola | Dettaglio |
|--------|-----------|
| Open source first | Tutto open source |
| Solo terminale | Niente interfaccia grafica — solo testo, ncurses, o output a caratteri |
| Linguaggi consentiti | Bash, Python, C, C++, Rust, Lua, Perl — tutti open source |
| Tema | Svelato all'inizio delle 2 ore |
| Presentazione | Si mostra il gioco nel terminale, possibilmente con effetti "hacker" |
